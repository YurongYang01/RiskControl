# 剩余开发计划：金融风控 CoT 工具链整合

根据目前进度，后端服务层与 API 路由层已基本完成。接下来的重点是**入口构建**与**前端可视化界面**开发，以实现“结构清晰、前端完整美观”的目标。

## 1. 核心入口与环境构建
- **创建启动入口 (`run.py`)**: 
  - 配置 Flask 应用启动参数，确保开发模式下的热重载。
- **完善依赖管理 (`requirements.txt`)**: 
  - 整合 `flask`, `pandas`, `openai`, `deepseek` 等所有依赖。

## 2. Web 路由层补全
- **实现页面路由 (`app/routes/main.py`)**: 
  - 负责渲染 HTML 模板，连接前端页面与后端 API。
  - 定义 `/`, `/generator`, `/inference`, `/inspector` 等页面访问入口。

## 3. 前端界面开发 (Tailwind CSS + Alpine.js)
采用现代轻量级技术栈，无需复杂构建工具即可实现响应式与交互。
- **基础布局 (`templates/base.html`)**: 
  - 统一的侧边栏导航、顶部栏。
  - 集成 Toast 通知系统（用于操作反馈）。
  - 引入 Tailwind CDN 和 Alpine.js。
- **数据工厂页 (`templates/generator.html`)**: 
  - **左侧**: 数据生成配置（样本数）、Prompt 模板配置。
  - **右侧**: 生成数据预览表格、Alpaca 格式预览。
- **推理引擎页 (`templates/inference.html`)**: 
  - **任务配置**: 模型选择、API Key 输入、并发数设置。
  - **实时监控**: 进度条、实时日志流（Server-Sent Events）、状态指示器。
- **质检工作台 (`templates/inspector.html`)**: 
  - **配置区**: 选择待质检文件、切换规则/模型模式。
  - **结果区**: 质检评分分布图、通过率统计、Bad Case 详情展示。

## 4. 集成验证与交付
- **全流程测试**: 
  - 模拟完整链路：生成数据 -> Prompt 转换 -> R1 推理 -> 质量检测。
- **启动服务**: 
  - 运行应用并提供预览链接。
